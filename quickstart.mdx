## Overview

This guide will walk you through creating a basic ZeroDash dashboard with application tiles, buttons, and search functionality.

<Steps>
  <Step title="Download ZeroDash">
    Clone the repository or download the source files
    
    ```bash
    git clone https://github.com/ujwalnk/zerodash.git
    cd zerodash
    ```
  </Step>
  
  <Step title="Create your configuration">
    Create a `user/config.js` file with your dashboard layout
    
    This file is gitignored by default so your personal configuration stays private.
  </Step>
  
  <Step title="Start a local server">
    Use any static file server to preview your dashboard
    
    ```bash
    python -m http.server 8080
    ```
    
    Then open `http://localhost:8080` in your browser.
  </Step>
</Steps>

## Your first configuration

Create `user/config.js` with this example configuration:

<CodeGroup>
```javascript user/config.js
export const dashboardConfig = {
  layouts: [
    {
      name: "desktop",
      rows: 3,
      columns: 4,
      tiles: [
        {
          type: "application",
          layout: { x: 0, y: 0, width: 1, height: 1 },
          config: {
            label: "Jellyfin",
            url: "https://jellyfin.local",
            image: "./icons/jellyfin.png",
            target: "same",
            availabilityUrl: "https://jellyfin.local/health"
          }
        },
        {
          type: "application",
          layout: { x: 1, y: 0, width: 1, height: 1 },
          config: {
            label: "Portainer",
            url: "https://portainer.local",
            image: "./icons/portainer.png",
            target: "new"
          }
        },
        {
          type: "link",
          layout: { x: 2, y: 0, width: 1, height: 1 },
          config: {
            label: "GitHub",
            url: "https://github.com",
            target: "new"
          }
        },
        {
          type: "button",
          layout: { x: 0, y: 1, width: 2, height: 1 },
          config: {
            label: "Restart Server",
            url: "https://homelab.local/api/restart",
            method: "POST",
            payload: { action: "restart" }
          }
        }
      ]
    }
  ]
};
```
</CodeGroup>

<Note>
  The configuration file must export a `dashboardConfig` object with a `layouts` array.
</Note>

## Understanding the structure

### Dashboard config

The root configuration object has one required key:

```javascript
{
  layouts: []  // Array of layout objects
}
```

### Layout object

Each layout defines a grid and its tiles:

| Property | Type | Description |
|----------|------|-------------|
| `name` | string | Unique identifier for this layout |
| `rows` | number | Number of grid rows |
| `columns` | number | Number of grid columns |
| `tiles` | array | Array of tile objects |
| `mediaQuery` | string (optional) | CSS media query for responsive layouts |

### Tile object

All tiles share this base structure:

```javascript
{
  type: "application",           // Tile type
  layout: {                      // Grid position
    x: 0,                        // Column (0-indexed)
    y: 0,                        // Row (0-indexed)  
    width: 1,                    // Columns to span
    height: 1                    // Rows to span
  },
  config: {                      // Tile-specific configuration
    // Configuration varies by tile type
  }
}
```

<Warning>
  Grid positions are **0-indexed**. A tile at `x: 0, y: 0` occupies the top-left cell.
</Warning>

## Application tiles

Application tiles are the most common tile type for linking to your homelab services:

```javascript
{
  type: "application",
  layout: { x: 0, y: 0, width: 1, height: 1 },
  config: {
    label: "Jellyfin",                              // Required: Display name
    url: "https://jellyfin.local",                  // Required: Target URL
    image: "./icons/jellyfin.png",                  // Optional: Icon path
    target: "same",                                  // Optional: "same" or "new" (default: "same")
    availabilityUrl: "https://jellyfin.local/health" // Optional: Health check endpoint
  }
}
```

<Info>
  When `availabilityUrl` is provided, ZeroDash will periodically check the endpoint and show a visual indicator if the service is unavailable.
</Info>

## Button tiles

Button tiles trigger HTTP requests when clicked:

```javascript
{
  type: "button",
  layout: { x: 0, y: 1, width: 2, height: 1 },
  config: {
    label: "Restart Container",               // Required: Button text
    url: "https://portainer.local/api/restart", // Required: Target URL
    method: "POST",                             // Required: "GET" or "POST"
    payload: {                                  // Optional: JSON payload (POST only)
      containerId: "abc123"
    },
    image: "./icons/restart.png"                // Optional: Button icon
  }
}
```

## Adding icons

ZeroDash references icons from the `icons/` directory, but this directory is gitignored. You can:

1. Create an `icons/` directory and add your own PNG/SVG files
2. Use external URLs: `https://cdn.example.com/icons/app.png`
3. Use data URLs for embedded icons
4. Omit the `image` property to show text-only tiles

## Responsive layouts

Define multiple layouts with media queries for different screen sizes:

<CodeGroup>
```javascript Desktop & Mobile
export const dashboardConfig = {
  layouts: [
    {
      name: "desktop",
      mediaQuery: "(min-width: 768px)",
      rows: 4,
      columns: 6,
      tiles: [
        // Desktop layout with 6 columns
      ]
    },
    {
      name: "mobile",
      // No mediaQuery = fallback layout
      rows: 8,
      columns: 2,
      tiles: [
        // Mobile layout with 2 columns
      ]
    }
  ]
};
```
</CodeGroup>

<Tip>
  Layouts are evaluated in order. The first matching media query wins. If no media queries match, the first layout without a `mediaQuery` is used as the fallback.
</Tip>

## Configuring search

Customize the global search feature by creating `user/searchConfig.js`:

<CodeGroup>
```javascript user/searchConfig.js
export const searchConfig = {
  enabled: true,
  
  // Customize prefix shortcuts
  prefixes: {
    '@': ['application'],        // @ = applications only
    '#': ['application', 'link'], // # = apps and links
    '/': ['button'],             // / = buttons only
    '.': ['toggle']              // . = toggles only
  }
};
```
</CodeGroup>

Press any key to open search, then use:
- `@jellyfin` to search only application tiles
- `/restart` to search only button tiles
- `server` to search all tile types

## Testing your dashboard

<Steps>
  <Step title="Start your local server">
    ```bash
    python -m http.server 8080
    ```
  </Step>
  
  <Step title="Open in browser">
    Navigate to `http://localhost:8080`
  </Step>
  
  <Step title="Test the search">
    Press any key to open the command palette and verify your tiles appear
  </Step>
  
  <Step title="Check the developer overlay">
    The developer overlay (visible by default) shows the current layout name and grid structure
  </Step>
</Steps>

<Warning>
  If you see a "Fatal Error" message, check the browser console for detailed validation errors. ZeroDash strictly validates configuration to catch errors early.
</Warning>
